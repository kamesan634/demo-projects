## 1. 系統概述

### 1.1 專案背景

本系統為一套針對一般零售業設計的簡易ERP（Enterprise Resource Planning）系統，旨在整合零售業日常營運所需的核心功能，包括商品管理、庫存控管、銷售作業、採購管理、客戶關係管理及報表分析等。

### 1.2 系統定位

| 項目 | 說明 |
|------|------|
| **目標用戶** | 中小型零售商店、連鎖門市、專賣店 |
| **系統類型** | Web-based 應用程式 |
| **部署方式** | 雲端部署 / 本地部署皆可 |
| **展示用途** | 作為技術能力展示與概念驗證 (POC) |

### 1.3 技術架構概述

```
┌─────────────────────────────────────────────────────────────┐
│                      前端 (Frontend)                         │
│              React / Vue.js + TypeScript                     │
├─────────────────────────────────────────────────────────────┤
│                      API 層 (RESTful)                        │
│                    Node.js / Express                         │
├─────────────────────────────────────────────────────────────┤
│                    後端 (Backend)                            │
│              Business Logic Layer                            │
├─────────────────────────────────────────────────────────────┤
│                    資料層 (Database)                         │
│              PostgreSQL / MySQL                              │
└─────────────────────────────────────────────────────────────┘
```

## 2. 專案範圍與目標

### 2.1 專案目標

| 目標類型 | 描述 |
|---------|------|
| **主要目標** | 建立一套功能完整、操作直覺的零售業ERP展示系統 |
| **業務目標** | 展示零售業核心流程的數位化解決方案 |
| **技術目標** | 展示現代化Web應用開發能力與系統整合能力 |

### 2.2 專案範圍

#### 包含範圍 (In Scope)

- 商品主檔管理
- 庫存管理與盤點
- POS銷售作業
- 採購進貨管理
- 供應商管理
- 客戶管理 (基礎CRM)
- 員工與權限管理
- 報表與統計分析
- 系統設定與參數維護

#### 排除範圍 (Out of Scope)

- 會計總帳系統
- 薪資計算系統
- 電子商務整合
- 多幣別處理
- 製造生產模組
- 物流配送追蹤

### 2.3 假設與限制

**假設條件**:
- 單一公司、可支援多門市
- 使用新台幣為主要幣別
- 標準稅率計算 (5% 營業稅)
- 網路環境穩定

**限制條件**:
- 展示系統，非正式生產環境
- 不處理實際金流
- 不與外部系統串接

## 3. 使用者角色與權限

### 3.1 角色定義

| 角色代碼 | 角色名稱 | 說明 |
|---------|---------|------|
| ADMIN | 系統管理員 | 擁有所有功能權限，負責系統設定與維護 |
| MANAGER | 店長/經理 | 管理門市營運，可存取報表與核准功能 |
| CASHIER | 收銀員 | 執行POS銷售作業，基本庫存查詢 |
| WAREHOUSE | 倉管人員 | 庫存管理、進貨驗收、盤點作業 |
| PURCHASER | 採購人員 | 採購單管理、供應商維護 |
| VIEWER | 檢視者 | 僅能檢視報表，無法異動資料 |

### 3.2 權限矩陣

| 功能模組 | ADMIN | MANAGER | CASHIER | WAREHOUSE | PURCHASER | VIEWER |
|---------|:-----:|:-------:|:-------:|:---------:|:---------:|:------:|
| 系統設定 | ✓ | - | - | - | - | - |
| 使用者管理 | ✓ | - | - | - | - | - |
| 商品管理 | ✓ | ✓ | R | R | R | R |
| 庫存管理 | ✓ | ✓ | R | ✓ | R | R |
| POS銷售 | ✓ | ✓ | ✓ | - | - | - |
| 採購管理 | ✓ | ✓ | - | R | ✓ | R |
| 供應商管理 | ✓ | ✓ | - | R | ✓ | R |
| 客戶管理 | ✓ | ✓ | ✓ | - | - | R |
| 報表查詢 | ✓ | ✓ | R | R | R | R |
| 盤點作業 | ✓ | ✓ | - | ✓ | - | - |

> ✓ = 完整權限 (CRUD), R = 僅讀取, - = 無權限

### 3.3 權限控制機制

```
權限檢查流程:
1. 使用者登入 → 取得 JWT Token (含角色資訊)
2. 發出 API 請求 → 中介層驗證 Token
3. 檢查角色權限 → 對照權限矩陣
4. 通過 → 執行操作 / 拒絕 → 回傳 403 Forbidden
```

### 3.4 系統管理功能

#### 3.4.1 使用者管理 (F01-001)

**功能說明**: 管理系統使用者帳號，包含新增、修改、停用、重設密碼等功能。

| 項目 | 說明 |
|------|------|
| 功能代碼 | F01-001 |
| 功能名稱 | 使用者管理 |
| 前置條件 | 使用者已登入且具有 ADMIN 角色 |
| 後置條件 | 使用者資料更新至資料庫 |

**輸入欄位**:

| 欄位名稱 | 資料型態 | 必填 | 驗證規則 |
|---------|---------|:----:|---------|
| 帳號 (username) | VARCHAR(50) | Y | 唯一值、英數字、4-50字元 |
| 密碼 (password) | VARCHAR(255) | Y | 最少8字元、含大小寫及數字 |
| 姓名 (full_name) | NVARCHAR(100) | Y | 2-100字元 |
| Email | VARCHAR(255) | Y | 有效Email格式 |
| 手機 (phone) | VARCHAR(20) | N | 台灣手機格式 |
| 角色 (role_id) | INT | Y | 必須為有效角色 |
| 所屬門市 (store_id) | INT | N | 必須為有效門市 |
| 狀態 (status) | ENUM | Y | ACTIVE / INACTIVE / LOCKED |

**業務規則**:
- BR01: 帳號建立後不可修改
- BR02: 密碼需加密儲存 (bcrypt)
- BR03: 連續登入失敗5次自動鎖定帳號
- BR04: 密碼90天強制更換提醒

#### 3.4.2 角色權限管理 (F01-002)

**功能說明**: 設定各角色可存取的功能模組與操作權限。

**輸入欄位**:

| 欄位名稱 | 資料型態 | 必填 | 說明 |
|---------|---------|:----:|------|
| 角色代碼 | VARCHAR(20) | Y | 唯一識別碼 |
| 角色名稱 | NVARCHAR(50) | Y | 顯示名稱 |
| 描述 | NVARCHAR(200) | N | 角色說明 |
| 權限清單 | JSON | Y | 功能權限對照 |

#### 3.4.3 系統參數設定 (F01-003)

**功能說明**: 維護系統運作所需的各項參數設定。

**參數類型**:

| 參數類別 | 參數項目 | 預設值 |
|---------|---------|--------|
| 公司資訊 | 公司名稱、統編、地址、電話 | - |
| 稅務設定 | 營業稅率 | 5% |
| 庫存設定 | 安全庫存警示天數 | 7天 |
| 銷售設定 | 發票格式、收據格式 | - |
| 密碼原則 | 密碼長度、複雜度、有效期 | 8/中/90天 |

#### 3.4.4 操作日誌 (F01-004)

**功能說明**: 記錄使用者所有重要操作，供稽核追蹤使用。

**記錄欄位**:

| 欄位 | 說明 |
|------|------|
| log_id | 日誌流水號 |
| user_id | 操作者ID |
| action_type | 操作類型 (CREATE/UPDATE/DELETE/LOGIN/LOGOUT) |
| module | 功能模組 |
| target_id | 操作對象ID |
| old_value | 修改前數值 (JSON) |
| new_value | 修改後數值 (JSON) |
| ip_address | 來源IP |
| created_at | 操作時間 |

#### 3.4.5 門市管理 (F01-005)

**功能說明**: 管理公司旗下各門市據點資訊。

**輸入欄位**:

| 欄位名稱 | 資料型態 | 必填 | 說明 |
|---------|---------|:----:|------|
| 門市代碼 | VARCHAR(20) | Y | 唯一識別碼 |
| 門市名稱 | NVARCHAR(100) | Y | - |
| 地址 | NVARCHAR(200) | Y | - |
| 電話 | VARCHAR(20) | Y | - |
| 負責人 | INT (FK) | N | 關聯使用者 |
| 營業時間 | JSON | N | 週一至週日營業時段 |
| 狀態 | ENUM | Y | ACTIVE / INACTIVE |

### 6.3 索引策略

**高頻查詢索引**:

```sql
-- 商品查詢
CREATE INDEX idx_products_sku ON products(sku);
CREATE INDEX idx_products_barcode ON products(barcode);
CREATE INDEX idx_products_category ON products(category_id);
CREATE INDEX idx_products_status ON products(status);

-- 庫存查詢
CREATE INDEX idx_inventory_product_warehouse
  ON inventory(product_id, warehouse_id);
CREATE INDEX idx_inventory_quantity ON inventory(quantity);

-- 訂單查詢
CREATE INDEX idx_orders_store_date ON orders(store_id, order_date);
CREATE INDEX idx_orders_customer ON orders(customer_id);
CREATE INDEX idx_orders_order_no ON orders(order_no);
CREATE INDEX idx_orders_status ON orders(status);

-- 會員查詢
CREATE INDEX idx_customers_phone ON customers(phone);
CREATE INDEX idx_customers_member_no ON customers(member_no);

-- 採購單查詢
CREATE INDEX idx_po_supplier ON purchase_orders(supplier_id);
CREATE INDEX idx_po_status ON purchase_orders(status);
```

### 7.2 通用回應格式

```json
{
  "success": true,
  "data": { ... },
  "meta": {
    "page": 1,
    "per_page": 20,
    "total": 100,
    "total_pages": 5
  }
}
```

**錯誤回應**:
```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "欄位驗證失敗",
    "details": [
      { "field": "email", "message": "Email格式不正確" }
    ]
  }
}
```

### 7.3 HTTP狀態碼

| 狀態碼 | 說明 | 使用情境 |
|:-----:|------|---------|

| 204 | No Content | 刪除成功 |
| 400 | Bad Request | 請求格式錯誤 |
| 401 | Unauthorized | 未認證 |
| 403 | Forbidden | 無權限 |
| 404 | Not Found | 資源不存在 |
| 409 | Conflict | 資源衝突 |
| 422 | Unprocessable Entity | 驗證失敗 |
| 500 | Internal Server Error | 伺服器錯誤 |
